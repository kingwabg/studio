# 디자인 브리프 — Document Studio (`/studio` 새 캔버스)

> 이 문서는 **UI 리디자인을 맡는 Claude(또는 디자이너)에게 넘기는 자료**다.
> 저장소 접근이 되면 이 문서 + 앱 실행(`npm run dev`)만으로 바로 작업을 시작할 수 있다.
> 작성: 2026-07-09.

---

## 1. 제품 한 줄
캔바(Canva)식 자유 배치 UI를 가진 **한국어 HWPX(한글) 호환 문서 편집기**.
최종 목표는 **월 구독 + 템플릿 마켓** (캔바 비즈니스 모델). 그래서 UI 품질이 곧 상품성이다.
지금은 기존 흐름 에디터(`/`, 무손상 유지)와 새 모듈형 캔버스(`/studio`)가 공존하며, **리디자인 대상은 `/studio`뿐**이다.

## 2. 지금 무엇이 문제인가 (오너 피드백)
> "밋밋하고 촌스럽다."

이건 색 몇 개의 문제가 아니라 **깊이(depth)·위계·마이크로 디테일의 부재**다. 구체적으로:
- **평면성**: 카드·패널·툴바가 전부 같은 평면에 얹혀 있어 레이어감이 없다. 그림자·경계·표면 대비가 약하다.
- **빈 공간 방치**: 홈은 광활한 여백에 카드 2개가 덩그러니. 밀도·리듬이 없다.
- **위계 약함**: 제목/부제/본문, 1차/2차 액션의 시각 무게 차이가 작다.
- **디테일 부재**: 호버·포커스·전환·아이콘 정렬 같은 "손맛"이 거의 없다.

## 3. 지향 방향 (오너가 판단 위임 → 아래로 확정)
- **베이스 = Linear / 노션급 "절제된 프로 미니멀"**: 채도 낮은 팔레트, 넉넉한 여백, 명확한 타입 스케일, 은은한 깊이(계층별 그림자 1~2단), 정교한 호버/포커스. 촌스러움의 반대는 화려함이 아니라 **정밀함**이다.
- **홈 / 템플릿 갤러리만 캔바식 따뜻함**: 구독·마켓이 목표라 첫인상은 친근하고 컬러풀하게. 문서 썸네일이 주인공.
- **편집기(에디터)는 도구답게 차분하게**: 지면이 주인공, 크롬(툴바·패널)은 뒤로 물러난다.

## 4. 절대 제약 (건드리면 회귀 — 반드시 지킬 것)
1. **기존 앱(`/`) 무손상**: `src/DocumentStudio.jsx`, `src/table-king/**` 는 손대지 않는다.
2. **Tailwind는 utilities-only** (preflight 제외 — `src/tailwind.css` 상단 주석 참고). 전역 리셋을 켜면 기존 앱 목록 불릿 등이 깨진다.
3. **Radix Themes는 `.radix-themes` 스코프**(`StudioTheme.tsx`)로 `/studio`에만. 밖으로 새면 안 됨.
4. **지면(`.canvas-dots`) 폰트 = 맑은 고딕 고정**. 한글 조판이 한글 글자를 전각(1em)으로 계산하는데 맑은 고딕이 정확히 1em이라 화면 줄바꿈 = 한글 줄바꿈이 된다. **바꾸면 미리보기·내보내기 정합이 깨진다.** (UI 셸 폰트 Pretendard는 자유롭게)
5. **A4 지면은 210×297mm, mm 좌표계**. 눈금자·스냅·좌표가 여기에 묶여 있다. 지면 자체의 비율/단위는 시각 장식이 아니라 기능이다.
6. **아이콘 = 인라인 SVG(1.4~1.6px 스트로크)만, 이모지 금지.** (`src/ui/icons.tsx`)
7. **한국어 UI**. 카피도 한국어.

## 5. 현재 디자인 토큰 (있는 그대로)
UI 셸은 아래 브랜드 색(`src/tailwind.css`)과 Radix Themes를 **혼용**하고 있다 — 이 이원화 자체가 정리 대상일 수 있다.

| 토큰 | 값 | 용도 |
|---|---|---|
| ink | `#1A2233` | 본문/제목 잉크 |
| inksoft | `#5B6577` | 보조 텍스트 |
| inkfaint | `#98A2B3` | 캡션/플레이스홀더 |
| accent (문서 블루) | `#2B5CE6` | 1차 액션·강조 |
| accenthover | `#1F49C4` | 액션 호버 |
| accentsoft | `#EDF2FE` | 강조 배경 |
| accentline | `#C4D4F9` | 강조 테두리 |
| line (헤어라인) | `#E4E8EF` | 기본 경계 |
| linestrong | `#CBD2DE` | 강한 경계 |
| paper | `#F6F7FA` | 카드/입력 배경 |
| canvas | `#EDF0F5` | 캔버스 바탕 |

- **폰트**: UI = Pretendard Variable / 지면 = 맑은 고딕(불변)
- **Radix 설정**(`StudioTheme.tsx`): `accentColor=indigo`, `grayColor=slate`, `radius=large`, `appearance=light`
- **홈 카드 호버**: `translateY(-2px)` + 2단 그림자 (`.click-card`)
- ⚠ 다크 모드 없음(현재 `appearance=light` 고정). 도입 여부는 오너 결정 필요.

## 7. 작업 규칙 (리디자이너용)
- 스타일은 `src/tailwind.css`(브랜드 유틸)와 Radix props로 낸다. 새 브랜드 색이 필요하면 `tailwind.css`의 레이어-밖 규칙 패턴을 따를 것(주석 참고).
- 컴포넌트 구조(props·상태·dnd-kit·zustand 셀렉터)는 **가능하면 유지**하고 표현(className/스타일)만 바꾼다. 특히 `CanvasBlock.tsx`는 로직이 촘촘하니 시각만 조정.
- **검증**: 바꾼 화면은 반드시 `npm run dev`로 실제 렌더 확인. 그리고 `/`(기존 앱)이 안 깨졌는지 1회 확인.
- 산출물은 화면 단위 PR/커밋으로 쪼갠다(홈 → 에디터 셸 → 패널 → …). 한 번에 다 갈아엎지 말 것.
