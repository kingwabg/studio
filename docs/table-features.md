## 발견·수정한 결함 (이번 점검에서)

### 1. 가져온 표가 원본 열 너비·행 높이를 잃음 → "표가 깨져 보임"의 주범
- **증상**: 한글 파일을 가져오면 모든 표가 균등 폭(약 530px 꽉 참)으로 변해 원본과 다르게 보임
- **원인**: importCore가 셀 텍스트·병합만 읽고 크기를 버림 (v1 한계)
- **수정**: rhwp `getCellProperties`로 셀별 너비/높이(HWPUNIT)를 읽어 px로 환산해 복원.
  병합 앵커의 크기는 스팬 구간에 균등 분배 ([importCore.js](../src/hwpx/importCore.js))
- **검증**: verify ⑦ "열 너비 복원" 게이트 (40/20/30mm → 151.2/75.6/113.4px) + 앱 확인

### 2. fold()가 표 크기 정보를 떨어뜨림
- **증상**: collectRaw가 읽은 widthsPx가 최종 JSON에 없음
- **원인**: 원시 블록 → 문서 JSON 접기 단계에서 rows/merges만 복사
- **수정**: widthsPx/heightsPx 통과시킴

### 3-1. 표 왼쪽(·위쪽) 테두리가 안 보임 (사용자 보고)
- **증상**: 표의 왼쪽 테두리가 렌더되지 않음 (환경에 따라 위쪽도)
- **원인**: 표의 모든 선은 `.boundary-layer`의 절대배치 세그먼트가 그리는데,
  세그먼트는 경계 "중심"에 그려져 외곽선의 절반(0.5px)이 프레임 밖에 있다.
  `.table-frame { overflow: hidden }`이 그 절반을 잘라 0.5px만 남고,
  DPR/줌 반올림에 따라 왼쪽·위쪽 선이 통째로 사라짐 (오른쪽·아래는 운 좋게 보였을 뿐 같은 문제)
- **수정**: `.table-frame`을 `overflow: visible`로 ([table-king.css](../src/table-king/table-king.css)).
  외곽 경계의 호버/드래그 하이라이트 절반 잘림도 함께 해결
- **검증**: 표 3개(활성/비활성) 전부에서 네 변 세그먼트 존재 + 클리핑 조상 없음 확인

### 3-2. 반올림으로 인한 1px 유령 열
- **증상**: 가져온 병합 표의 colgroup에 1px짜리 열이 생김 (경계 핸들이 지저분해짐)
- **원인**: 행마다 따로 `Math.round`하면 같은 경계(30+30 vs 40+20 = 60mm)가 1px 어긋남
- **수정**: 반올림 제거 — 같은 mm에서 온 float은 행끼리 정확히 일치해 경계가 합쳐짐
  ([DocumentStudio.jsx](../src/DocumentStudio.jsx) hydrateImported)

## 알려진 한계 (다음 과제 후보)

1. **가져오기 시 셀 스타일(배경·글자색·굵게) 미복원** — 크기·병합·내용까지만.
   rhwp `getCellCharPropertiesAt`/`getCellStyleAt`으로 확장 가능
2. **한 페이지보다 큰 표** — 페이지 경계에 걸친 채 유지 (표 행 분할은 별도 과제)
3. **중첩 표·이미지 셀** — 가져오기에서 무시됨
4. **클립보드** — 브라우저 정책상 포커스 있는 창에서만 동작 (앱 문제 아님)
