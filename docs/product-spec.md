## 로드맵 (우선순위 정본)

### P0 — 캔버스 한컴 완성 (현 스프린트)
1. 캔버스 모드 ② 새 문서 기본 여백 0 (화면 좌표 = 내보내기 원점 일치)
2. ③ 다중 선택·정렬·복제 (마퀴, Ctrl+D — rhwp 다중 그림 선택을 표까지 확장)
3. ④ 크롬 다이어트 (캔버스 모드에서 한컴 메뉴/툴바 접고 선택 컨텍스트 툴바)
4. ⑤ 레이어 패널 + z순서 · ⑥ 템플릿 갤러리 (마켓의 씨앗)

### P1 — AI 심화 (차별점 강화)
5. 표 셀 선택 우클릭 "AI에게 수정하기" (글상자와 동일 UX)
6. AI 수정 전/후 단어 단위 diff 하이라이트 (inline-ai의 "한눈에 비교")
7. 자료 읽어 채우기: 엑셀/CSV를 주면 표에 부어넣기 (/studio 병합 로직 재이용)

### P2 — 사업화 게이트 (배포 전 필수)
8. AI 프록시 서버리스 이관 (현 dev 전용 — Vercel Functions)
9. 저장/회원 (Supabase — 단 NAS 수요 대비 repository 인터페이스 유지)
10. 결제 (토스페이먼츠/포트원, 글로벌 Stripe) → 월 구독
11. 템플릿 마켓 (갤러리 ⑥ 위에 공유·판매)

### P3 — 킬러 기능 백로그 (사용자 기획 6종)
12. 지자체 공문 양식 리버스 (빈 HWPX 신청서 → AI가 입력 가능 템플릿화)
13. 데이터 다중 병합 rhwp 이식 (출석부·명단 → 수십 명치 자동 생성)
14. 음성→텍스트 회의록 (STT + 화자 구별)
15. 공사/현장 Before&After 사진 스마트 그리드 (AI crop·표 규격 배치)
16. 블록 단위 코멘트·반려 핀 (Figma식 — 결재 반려 직관화)
17. 내부 서버/NAS 저장 + 자동 폴더 분류

## AI 조언 의무 — 제안의 기준 (운영 규칙은 playbooks/agent-protocol.md §6)

- **기능 제안**: 이 로드맵과 "차별점 3종"에 부합하는 것만. 제안 형식 = "무엇을(1줄) + 로드맵
  어디에 끼우나(P0~P3) + 왜 지금(1줄)". 로드맵에 없는 신규 아이디어는 P 배정을 제안하고
  사용자 승인 후 이 문서에 추가.
- **라이브러리 추천**: docs/tech-choices.md(기술 선택 대장) 먼저 대조 — 이미 결정된 영역은
  재론 금지(재검토 트리거 충족 시만). 미결 영역은 후보 1~3개를 [기능 적합성 · 유지보수 활발성 ·
  라이선스(MIT/Apache — 저작권 Safe 전략과 동결) · 번들 영향]으로 비교해 선택지로 제시.
- **구현 가이드**: 채택 시 단계별 계획(파일 단위 + 검증 방법 + 완료 게이트)을 코드 작성 전에 제시.
