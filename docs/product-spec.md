**한 줄 비전**: 저작권 걱정 없는(Safe), AI가 통합된, HWPX(한글) 완전 호환 캔바식 문서 도구.
**사업 모델**: 월 구독 + 템플릿 공유 마켓. **타깃**: 공무원·관공서·센터·현장 실무자.
**차별점 3종**: ①관공서 한글에서 그대로 열리는 HWPX ②폰트 저작권 100% 안전 ③캔바식 자유 배치+AI.

이 문서는 로드맵의 정본이다. 우선순위 변경·기능 추가는 여기 먼저 반영(+playbooks/CHANGELOG.md).
상세 아키텍처는 CLAUDE.md, 과거 서사는 docs/rhwp-adoption.md.

## 아키텍처 원칙 (2026-07-12 — Inline AI 비교 분석에서 확정)

경쟁(Inline AI) = 설치된 한글을 원격 조종하는 로컬 RPA 에이전트. **우리 = 에디터 그 자체(웹)** —
조종할 외부 프로그램이 없으므로 RPA 계층이 통째로 불필요(구조적 우위). 로컬 에이전트 노선은
별도 프로젝트(junha-ai)의 영역이며 이 저장소에 섞지 않는다.
1. **로컬 퍼스트**: 문서는 브라우저 안에서 처리(현재 서버 저장 없음). 저장(P2-9) 도입 후에도
   "AI엔 문서 전체가 아니라 필요한 맥락만" 원칙 유지.
2. **최소 맥락 전송**: AI 호출마다 전송 범위를 명시적으로 정의(선택 텍스트·대상 표만) —
   문서 전체 덤프 금지. 전송 내용 목록은 P2-8 데이터 처리 고지와 1:1 대응.
3. **승인 루프(HIL)**: AI의 문서 변경은 계획/전후 비교 → 사용자 승인 → 단일 스냅샷 적용
   (Ctrl+Z 일괄 취소). 이미 구현된 이 패턴이 **모든 신규 AI 기능의 기본**이다(다단계 작업 포함).

## 로드맵 (우선순위 정본)

### P0 — 캔버스 한컴 완성 (현 스프린트)
1. 캔버스 모드 ② 새 문서 기본 여백 0 — **보류** (준하 님 지시: 캔버스도 여백 유지 필요)
2. ③ 다중 선택·정렬·복제 (마퀴, Ctrl+D — rhwp 다중 그림 선택을 표까지 확장)
3. ④ 크롬 다이어트 (캔버스 모드에서 한컴 메뉴/툴바 접고 선택 컨텍스트 툴바)
4. ⑤ 레이어 패널 + z순서 · ⑥ 템플릿 갤러리 (마켓의 씨앗)
- **도그푸딩 게이트** (2026-07-12): P0 항목 하나 완료할 때마다 실문서 1종(회의록/공문 1장)을
  캔버스 한컴으로 처음부터 작성 → HWPX 내보내기 → **실제 한컴 한글에서 열어 확인**, 결과를
  rhwp-adoption.md에 3줄 기록. (AI가 못 하는 유일한 검증 — 사용자 루틴, 회당 ~10분)

### P1 — AI 심화 (차별점 강화)
5. 표 셀 선택 우클릭 "AI에게 수정하기" (글상자와 동일 UX)
6. AI 수정 전/후 단어 단위 diff 하이라이트 (inline-ai의 "한눈에 비교")
7. 자료 읽어 채우기: 엑셀/CSV를 주면 표에 부어넣기 (/studio 병합 로직 재이용)
8. AI 컨텍스트 미니마이저: 기능별 전송 범위 정의표 + AI 패널에 "이번에 AI로 보낸 내용" 표시
   (아키텍처 원칙 2의 구현 — Inline AI의 ZDR 마케팅에 대한 우리 답)

### P2 — 사업화 게이트 (배포 전 필수 — 실행 체크리스트는 docs/saas-gates.md G1~G10)
8. AI 프록시 서버리스 이관 (현 dev 전용 — Vercel Functions). 이관 시 **AI 데이터 처리 고지**
   동반 — 전송 대상(MiniMax·국외)/내용/보존을 처리방침·AI 패널 첫 사용 안내에 명시
9. 저장/회원 (Supabase — 단 NAS 수요 대비 repository 인터페이스 유지)
   - 9-1. **법정 문서 3종**: 개인정보처리방침(회원 오픈과 동시 필수)·이용약관·통신판매업
     신고(마켓 판매 개시 전)
   - 9-5. **파일럿 1명**: 결제 착수 전, 실무자(공무원/행정) 1명이 실업무 문서 1건을 완주 —
     막힌 지점을 P0/P1에 반영
10. 결제 (토스페이먼츠/포트원, 글로벌 Stripe) → 월 구독
11. 템플릿 마켓 (갤러리 ⑥ 위에 공유·판매)
   - 11-5. **릴리스 기본기**: 에러 수집 도구·git tag vX.Y.Z·앱 버전 표기 (결제 전 완료)

### P3 — 킬러 기능 백로그 (사용자 기획 6종)
12. 지자체 공문 양식 리버스 (빈 HWPX 신청서 → AI가 입력 가능 템플릿화)
13. 데이터 다중 병합 rhwp 이식 (출석부·명단 → 수십 명치 자동 생성)
14. 음성→텍스트 회의록 (STT + 화자 구별)
15. 공사/현장 Before&After 사진 스마트 그리드 (AI crop·표 규격 배치)
16. 블록 단위 코멘트·반려 핀 (Figma식 — 결재 반려 직관화)
17. 내부 서버/NAS 저장 + 자동 폴더 분류

## AI 조언 의무 — 제안의 기준 (운영 규칙은 playbooks/agent-protocol.md §4 기능 조언 의무)

- **기능 제안**: 이 로드맵과 "차별점 3종"에 부합하는 것만. 제안 형식 = "무엇을(1줄) + 로드맵
  어디에 끼우나(P0~P3) + 왜 지금(1줄)". 로드맵에 없는 신규 아이디어는 P 배정을 제안하고
  사용자 승인 후 이 문서에 추가. **신규 아이디어의 기본 착지는 P3(백로그)** — P0 진입은 기존
  P0 항목 1개 완료 또는 명시적 제외와 맞교환(1-in-1-out). 다음 착수 후보 제안은 P0 최상단
  미완 항목을 기본으로.
- **라이브러리 추천**: 절차·비교 기준의 정본 = docs/tech-choices.md "라이브러리 채택 절차" 절
  (여기 재서술 금지 — H4).
- **구현 가이드**: 채택 시 단계별 계획(파일 단위 + 검증 방법 + 완료 게이트)을 코드 작성 전에 제시.
