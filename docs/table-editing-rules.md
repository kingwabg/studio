# 표 편집 규칙

이 문서는 `/studio/editor`의 표 동작을 정의하는 기준서다. 표 UI를 수정할 때는
이 문서의 상태 전이, 입력 우선순위, 데이터 불변식을 먼저 확인한다.

## 1. 핵심 원칙

1. 표 데이터의 진실은 `Block.data`다.
   - `cells`: 셀 내용과 셀 서식
   - `widths`: 열 너비
   - `cellHeights`: 행 높이
   - `merges`: 병합 영역
2. 표의 전체 폭과 높이는 `widths`와 `cellHeights`의 합에서 파생한다. 표 블록의
   `w`와 `h`를 별도로 직접 수정해 렌더링하지 않는다.
3. 표 전체 크기를 조절해도 셀 안의 글꼴 크기와 글자 자체의 크기는 바꾸지 않는다.
   필요한 경우 열 너비와 행 높이만 재분배한다.
4. A4 본문 여백 밖으로 표가 이동하거나 커지지 않는다. 이동·리사이즈의 최종 좌표는
   여백 경계에서 클램프한다.
5. 표 엔진 원본의 구조와 내보내기 계약을 우선 보존한다. 시각 변경은 스코프된 CSS,
   주변 동작은 래퍼에서 해결한다.

## 2. 표 상태

### 2.1 객체 선택 모드

표를 선택했지만 셀을 편집하지 않는 상태다.

- 표 바깥 또는 표의 외곽 테두리를 클릭하면 표 객체가 선택된다.
- 표 안쪽의 내부 열·행 경계선은 이 상태에서 작동하지 않는다.
- 표 전체에 8개 외부 핸들을 표시한다.
  - 좌상, 상, 우상, 좌, 우, 좌하, 하, 우하
  - 핸들은 표 프레임에 직접 붙고 표 바깥쪽에 뜨지 않는다.
- 외부 핸들 드래그는 표 전체 크기 조절이다.
  - 우 핸들: 오른쪽으로 폭 조절
  - 하 핸들: 아래로 높이 조절
  - 우하 핸들: 폭과 높이를 동시에 조절
  - 나머지 핸들: 해당 축을 조절하되 A4 여백과 최소 크기를 지킨다.
- 표 표면을 드래그하면 표 전체 이동이다. 셀 텍스트나 내부 경계선의 드래그와
  충돌시키지 않는다.
- 객체 선택 모드에서는 내부 셀 경계선 resize hitbox를 비활성화한다.
- 객체 선택 표시에는 표 전체의 반투명 파란 오버레이를 사용하지 않는다. 외부
  핸들과 최소한의 선택 표시만 보여준다.

### 2.2 셀 편집 모드

표 안의 셀을 선택하고 내용을 입력·수정하는 상태다.

- 셀 텍스트를 클릭하면 해당 셀이 선택되고, 클릭 위치에 캐럿을 둔다.
- 셀의 빈 영역을 클릭해도 해당 셀의 마지막 글자 뒤에 캐럿을 둔다.
- 표에 이미 객체 선택 표시가 있어도 셀 클릭은 셀 편집 기준으로 처리한다.
- `Enter`는 선택 셀의 텍스트 편집으로 진입한다.
- `Escape`는 셀 편집을 종료하고 표 객체 선택 상태로 돌아간다.
- 셀 텍스트를 드래그하면 셀 범위 선택이다. 이 동작이 표 이동으로 해석되면 안 된다.
- 객체 모드용 외부 8개 핸들은 셀 편집 중 숨긴다.

### 2.3 상태 우선순위

포인터가 겹치는 경우 아래 순서로 처리한다.

1. 외부 크기 조절 핸들
2. 셀 내부 경계선 resize hitbox
3. 셀 텍스트 편집 영역
4. 표 이동 표면
5. 표 객체 선택
6. 캔버스 마퀴 선택 또는 배경 클릭

드래그가 시작되면 `pointer capture`로 시작한 조작의 대상과 커서를 고정한다.
드래그 중 다른 DOM 요소의 hover가 바뀌어도 조작 종류가 바뀌면 안 된다.

## 3. 표 전체 크기 조절

- 객체 선택 모드에서만 외부 8개 핸들을 활성화한다.
- 핸들 hitbox는 최소 12px로 유지하되, 보이는 핸들은 작고 일관된 원형 또는 사각형으로
  표시한다.
- 최소 열 너비는 30px. 행 높이의 하드 바닥은 24px이되, **콘텐츠 하한이 우선**한다:
  한글은 빈 셀에도 문단 부호 한 줄이 있으므로 빈 셀 최소 = `ceil(글자 12.5px × 160% + 상하
  안 여백)` (기본 여백에서 28px, 인스펙터 여백에 따라 커짐). 텍스트 셀 최소 =
  `max(한 줄 최소, 줄 수 × 줄높이 15px) + 상하 여백` (명시적 줄바꿈만 계산).
  단일 소스: `useBoundaryDrag.js`의 `minHeightForCell`/`buildContentMinRows`/`HANGUL_MIN_ROW_H`.
- 전체 축소 시 글자가 잘리지 않도록 각 열과 행의 최소값을 먼저 계산한다. 이 하한은
  **모든 경로 공통**이다: 내부 경계 드래그(일반·Shift), 외곽 8핸들, 대각선, 키보드 미세 조절,
  H 같게, 셀 나누기(트랙이 최소 미만으로 쪼개지면 최소로 고정하고 표가 자란다 — HWP식).
- 이미 하한 미만인 행(과거 데이터·줄 수 증가)은 "악화만 금지"한다 — 드래그 방향과 반대로
  강제 수리(점프)하지 않는다.
- 전체 확대·축소 중에는 표 프레임, 셀 배경, 셀 텍스트, 선택 표시가 동일한 좌표 계산을
  사용해야 한다. DOM의 지연된 별도 배경 박스를 만들지 않는다.
- 표 전체 resize는 열·행 비율을 유지하는 방식이 기본이다. 사용자가 내부 경계선을
  조절할 때는 비율을 유지하지 않고 해당 경계만 조절한다.
- 리사이즈 중에는 글꼴 크기, 줄간격, 셀 안쪽 여백을 자동으로 변경하지 않는다.
- 마우스가 핸들 위에 있을 때만 resize cursor를 표시한다. 핸들 밖에서는 표 이동
  cursor 또는 기본 cursor를 사용한다.

## 4. 셀 경계선 조절

### 4.1 Hover

- 셀 편집 모드에서만 내부 열·행 경계선 hover를 활성화한다.
- 내부 경계선 hitbox는 보이지 않으며 12px 폭 또는 높이를 사용한다.
- `Shift`가 없으면 해당 경계선의 전체 행 또는 전체 열을 파란색으로 강조한다.
- `Shift`를 누르면 마우스가 위치한 해당 셀 구간만 강조한다.
- 오른쪽 외곽 세로선은 일반 드래그에서는 표 전체 폭 조절 대상이다.
- 오른쪽 외곽 세로선을 `Shift`로 드래그할 때만 고정 앵커로 취급해 마지막 열 또는
  표 전체 구조가 깨지지 않게 한다.
- hover 선의 강조 상태는 얇은 단일 선으로 표시한다. 셀 전체에 반투명 배경을 칠해
  깜빡이는 것처럼 보이게 하지 않는다.

### 4.2 Drag

- 열 경계 드래그는 해당 열 너비를 변경한다. 콘텐츠 최소 폭(텍스트 실측) 아래로 못 줄인다.
- 행 경계 드래그는 **행 전체를 균일하게** 변경한다("한 행 = 한 높이" 불변식).
  셀은 div 절대배치라 열별 높이가 갈라지면 렌더가 그대로 노출된다 — 원본 `<table>`의
  `<tr>` 통일을 이제 드래그가 직접 보장한다(`moveTableRowBoundary`).
- `Shift` 드래그는 한 셀 구간만 조절한다(한컴식 어긋남). 병합 셀(cs>1)의 경계는 **스팬의
  모든 열을 한 그룹으로** 움직인다 — 렌더는 앵커 열을 읽으므로 단일 열만 쓰면 화면이 죽는다.
  고정해야 하는 앵커 경계(바닥 외곽 등)는 잠금 로직이 거부한다.
- 최소값은 §3의 콘텐츠 하한 규칙을 따른다(30px/24px는 하드 바닥일 뿐).
- 드래그 중 선택 셀, 배경, 테두리, 표 프레임을 같은 상태값에서 렌더링한다.
- pointermove마다 무거운 전체 컴포넌트를 재마운트하지 않는다. 시각 업데이트는
  `requestAnimationFrame` 단위로 묶고, 종료 시 한 번 저장한다.
- hover 후보가 인접 경계 사이에서 바뀌는 현상을 막기 위해 동일 경계 유지 우선,
  짧은 hysteresis 또는 안정화 지연을 사용한다.

## 5. 셀 선택과 편집

- 일반 클릭: 한 셀 선택
- 드래그: 연속 셀 범위 선택
- `Shift` 클릭: 현재 셀부터 클릭 셀까지 연속 범위 확장
- 병합된 셀은 논리적 하나의 셀처럼 선택하고 경계를 병합 영역에 맞춘다.
- 빈 셀도 선택 가능하며, 텍스트 캐럿은 마지막 글자 위치로 이동한다.
- 셀 선택 배경은 선택 범위에만 표시한다. 표 전체를 반투명 파란색으로 덮지 않는다.
- 셀 선택과 내부 경계 hover가 동시에 있을 때는 셀 선택 배경보다 경계선 강조가
  우선하지만, 선택 범위 자체는 계속 유지한다.

## 6. 표 이동과 선택 충돌 방지

- 표 객체 선택 상태에서 표 표면 또는 외곽 테두리 드래그는 표 이동이다.
- 셀 편집 상태에서 셀 텍스트 드래그는 텍스트 또는 셀 범위 선택이다.
- 셀 편집 상태에서 내부 경계선 드래그는 열·행 크기 조절이다.
- 외부 크기 조절 핸들에서 시작한 드래그는 항상 전체 크기 조절이다.
- 드래그 시작 후에는 시작한 조작을 변경하지 않는다. 예를 들어 내부 경계선에서
  시작한 드래그가 표 이동으로 바뀌면 안 된다.
- `mousedown`/`pointerdown` 전파 차단은 조작 대상별로 적용하고, 캔버스 전체의 선택
  해제 이벤트를 무조건 실행하지 않는다.

## 7. 표 기능과 리본

### 공통

- 실행 취소
- 다시 실행

### 셀 편집에서만 표시

- 행 추가
- 열 추가
- 행 삭제
- 열 삭제
- 셀 병합
- 셀 나누기
- 복사
- 잘라내기
- 붙여넣기
- 지우기
- 셀 가로 정렬: 좌·중·우
- 셀 세로 정렬: 상·중·하
- 테두리
- 배경색
- 열 너비 같게 (`W 같게`)
- 행 높이 같게 (`H 같게`)

표를 선택하지 않았을 때 표 전용 기능을 리본에 노출하지 않는다. 표 전용 기능과
텍스트 전용 기능을 같은 그룹에 중복해서 배치하지 않는다.

## 8. 우클릭 메뉴

셀 편집 모드에서 우클릭 메뉴는 구조 기능을 제공한다.

- 복사
- 붙여넣기
- 행 추가
- 열 추가
- 행 삭제
- 열 삭제
- 셀 병합
- 셀 나누기
- 테두리
- 배경색

리본은 자주 쓰는 스타일과 실행 취소를 담당하고, 우클릭은 현재 선택 범위에 대한
구조 작업을 담당한다. 같은 기능이 두 곳에 있어도 호출하는 데이터 경로는 하나여야
한다.

### 브라우저에 양보하는 단축키

아래 단축키는 표 안에서도 가로채지 않는다. 브라우저 탭·주소창·검색·인쇄·개발자
도구의 기본 동작을 보존한다.

- `Ctrl/Cmd+N`: 새 창
- `Ctrl/Cmd+T`: 새 탭
- `Ctrl/Cmd+W`: 탭 닫기
- `Ctrl/Cmd+L`: 주소창
- `Ctrl/Cmd+F`: 페이지 찾기
- `Ctrl/Cmd+P`: 인쇄
- `Ctrl/Cmd+S`: 페이지 저장
- `Ctrl/Cmd+R`: 새로고침
- `Ctrl/Cmd+F5`: 강력 새로고침
- `Ctrl/Cmd+Tab`: 탭 이동
- `Ctrl/Cmd+Shift+I`: 개발자 도구
- `Ctrl/Cmd+U`: 소스 보기

`F5`만 셀 편집 중에 한해 표 단축키로 사용한다. 표가 활성화되지 않았거나 셀 편집
중이 아니면 브라우저의 새로고침을 막지 않는다.

## 10. 클립보드 규칙

1. 표 내부 복사는 선택 범위의 셀 구조와 텍스트를 함께 만든다.
2. 외부 Excel·Google Sheets에서 붙여넣을 때는 `text/html`의 `<table>`을 먼저 파싱한다.
3. HTML 표가 없으면 `text/plain`의 TSV를 행·열 구조로 파싱한다.
4. 붙여넣기 범위가 현재 선택보다 크면 필요한 행·열을 자동으로 추가한다.
5. 병합 정보가 없는 외부 표는 병합하지 않고 일반 셀로 붙인다.
6. 붙여넣기 전후에 열 최소 너비·행 최소 높이·A4 본문 폭을 검증한다.

## 11. 선택 표시와 hover 표시

- 셀 선택: 선택 셀 범위에만 연한 파란 배경과 얇은 테두리.
- 내부 경계 hover: 얇은 파란 선 하나. 전체 표 배경은 변경하지 않는다.
- 객체 선택: 외부 8개 핸들 + 이동 가능한 표면. 내부 셀 선택 배경은 표시하지 않는다.
- 리사이즈 중: 현재 표 프레임, 셀 배경, 텍스트, 선택 표시가 함께 이동한다.
- 선택 표시와 실제 hitbox를 분리한다. 보이지 않는 hitbox를 넓혀도 보이는 선의
  두께와 색은 바꾸지 않는다.
- hover 상태는 포인터가 선 위를 이동하는 동안 깜빡이지 않아야 한다. 후보 경계가
  바뀔 때만 상태를 변경한다.

## 12. 수정 시 반드시 지킬 것

### 수정 전

- `git status`와 현재 브랜치를 확인한다.
- 변경 대상이 `/studio`인지 기존 `/` 편집기인지 확인한다.
- `Block.data` 구조와 `SCALE = 96 / 25.4`를 확인한다.
- 표 엔진의 원본 파일을 직접 고치지 않고 래퍼 또는 스코프 CSS에서 해결할 수 있는지
  먼저 판단한다.

### 수정 중

- **클래스명은 Tailwind 유틸리티명과 겹치면 안 된다** (`table-row`·`table-cell`·`grid`·
  `hidden` 등 금지 — utilities-only 주입이라 display가 덮여 레이아웃이 깨진다). 표 내부는
  `tk-` 접두사 또는 겹치는 이름에 `display` 명시(table-king.css의 방어 주석 참고).
- 병합 판정은 `table/merge.js`(coveringMerge/isCoveredByMerge/mergeAt) 단일 소스만 쓴다.
  예외: exportCore.js는 의존성 0 계약으로 자체 복사본 유지(상호 참조 주석).
- 셀 글자 치수는 `table/constants.js`(CELL_FONT_SIZE 12.5px·CELL_LINE_HEIGHT)가 단일 소스 —
  화면 inline·최소값 공식·내보내기 pt(×0.75)·스냅샷이 전부 여기서 파생. 재선언 금지.
- 포커스/캐럿 지연은 `setTimeout(0)`을 쓴다 — rAF는 백그라운드 탭에서 정지해 셀 이동 후
  포커스가 영영 안 따라오는 사고가 난다.
- 키보드 셀 이동(Tab·방향키·확장 모드)은 병합 스팬을 건너뛴다: 앵커에서 나갈 땐 `cs/rs`만큼,
  착지가 덮인 칸이면 그 병합의 앵커로. 덮인 좌표에 선택이 서면 포커스가 이전 셀에 남는
  탈선이 난다.
- 실행취소 히스토리는 **데이터**(cells/widths/heights/merges) 변경에만 쌓는다 — 선택·스피너
  변화로 밀어내지 않는다. 외곽 리사이즈·AI 적용 등 외부 변경도 히스토리에 들어가야
  Ctrl+Z가 그것까지 한 단계로 되돌린다.
- 외곽 리사이즈 커밋은 위치+데이터를 `setTableData(id, data, pos)` 한 번으로 — 나눠 커밋하면
  히스토리가 갈라지고 낡은 폭으로 좌표가 클램프된다.
- 저장 데이터 수리 규칙(로드 시): 열 합이 같으면 온전(의도적 어긋남 보존, 한 줄 최소 미만
  균일 행만 상향), 열 합이 다르면 손상 → 행별 최대로 통일.
- 표 전체 크기와 셀 경계 크기 조절을 같은 핸들로 합치지 않는다.
- 객체 선택 모드와 셀 편집 모드의 이벤트 전파를 분리한다.
- `pointermove`마다 표 상태를 새로 초기화하거나 DOM을 재마운트하지 않는다.
- 렌더링용 표 프레임과 저장용 `Block.data`의 좌표 계산을 다르게 만들지 않는다.
- 글자 크기를 표 리사이즈에 연동하지 않는다.
- 오른쪽 외곽 세로선의 일반 드래그와 `Shift` 드래그 규칙을 혼동하지 않는다.
- 표 바깥까지 전역 키보드 이벤트를 적용하지 않는다.

## 13. 현재 구현과의 차이

현재 코드에 이미 있는 항목은 유지하고, 아래 항목은 이 기준서에 맞춰 후속 보완한다.

- 표 객체 선택 모드의 외부 8개 핸들 동작과 표 이동 표면
- 셀 편집 중 내부 경계선 12px hitbox 및 일반/`Shift` hover 구분
- F5의 현재 셀 확장 → 전체 셀 확장 단계
- HTML 우선/TSV fallback 붙여넣기 계약
- hover 깜빡임 방지를 위한 안정화 상태 관리
- 표 전체 대각선 리사이즈에서 프레임·배경·선택 표시 동기화
- `Ctrl/Cmd+A` 1차 표 전체 선택과 2차 문서 선택의 명확한 상태 관리

이 차이 목록은 구현 완료 여부를 판단하는 체크리스트로 사용한다.
